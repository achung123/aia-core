## Findings

### [MEDIUM] Skeleton animation references undefined `@keyframes pulse`

**File:** `frontend/src/pages/AnalyticsPage.tsx`
**Line(s):** 118
**Category:** correctness

**Problem:**
The `skeleton` style uses `animation: 'pulse 1.5s ease-in-out infinite'` but no `@keyframes pulse` is defined in any stylesheet. The global `style.css` only defines `@keyframes spin`. The loading skeleton will render as a static gradient bar with no visual animation.

**Code:**
```typescript
skeleton: {
    height: 80,
    borderRadius: 8,
    background: 'linear-gradient(90deg, #1e1b4b 25%, #252250 50%, #1e1b4b 75%)',
    animation: 'pulse 1.5s ease-in-out infinite',
},
```

**Suggested Fix:**
Either add a `@keyframes pulse` definition to `frontend/src/style.css` (e.g., opacity 0.6 ↔ 1.0), or remove the `animation` property to match `AwardsGridPage`'s static skeleton approach.

**Impact:** The loading skeleton appears as a non-animated static block, reducing perceived responsiveness during data fetches.

### [LOW] Player name not URL-encoded in Link

**File:** `frontend/src/pages/AnalyticsPage.tsx`
**Line(s):** 45
**Category:** correctness

**Problem:**
The player profile link uses an unencoded player name: `` to={`/players/${entry.player_name}`} ``. If a player name contains characters like `/`, `#`, or `?`, navigation could break. Note: this is a pre-existing pattern also present in `AwardsGridPage.tsx` L65 — not a regression.

**Code:**
```tsx
<Link to={`/players/${entry.player_name}`} style={styles.playerLink}>
```

**Suggested Fix:**
Use `encodeURIComponent`: `` to={`/players/${encodeURIComponent(entry.player_name)}`} ``

**Impact:** Low risk — player names in poker are typically simple strings, and react-router handles most common characters. A future hardening pass could fix this globally.
