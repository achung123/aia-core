## Findings

### [LOW] Missing server_default on phase and action_index columns

**File:** `alembic/versions/72219689ea6b_add_hand_states_table.py`
**Line(s):** 27-31
**Category:** design

**Problem:**
`phase` (NOT NULL, default 'preflop') and `action_index` (default 0) have Python-side defaults only. Direct SQL inserts outside SQLAlchemy (e.g., manual DB operations, other tools) will not receive these defaults — `phase` would fail the NOT NULL constraint, and `action_index` would be NULL.

**Code:**
```python
sa.Column('phase', sa.String(), nullable=False),
sa.Column('action_index', sa.Integer(), nullable=True),
```

**Suggested Fix:**
Add `server_default` to the migration and model columns:
```python
# Migration
sa.Column('phase', sa.String(), nullable=False, server_default='preflop'),
sa.Column('action_index', sa.Integer(), nullable=True, server_default='0'),
# Model
phase = Column(String, nullable=False, default='preflop', server_default='preflop')
action_index = Column(Integer, default=0, server_default=text('0'))
```

**Impact:** Very low risk — the project uses SQLAlchemy exclusively and all other models follow the same Python-side-only default pattern. This is a defense-in-depth suggestion, not a correctness issue. Matches existing codebase convention.

### [LOW] Test file uses manual setup/teardown instead of pytest fixtures

**File:** `test/test_hand_state_model.py`
**Line(s):** 12-28
**Category:** convention

**Problem:**
The test file creates its own engine, `SessionLocal`, and `_setup()`/`_teardown()` functions instead of using pytest fixtures. Other model tests in the codebase (e.g., `test_hand_model.py`) use `@pytest.fixture` with `yield` for the same pattern, and `conftest.py` provides an `autouse` fixture that handles `create_all`/`drop_all`.

**Code:**
```python
engine = create_engine(
    'sqlite:///:memory:',
    connect_args={'check_same_thread': False},
    poolclass=StaticPool,
)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def _setup():
    Base.metadata.create_all(bind=engine)
    return SessionLocal()

def _teardown(db):
    db.close()
    Base.metadata.drop_all(bind=engine)
```

**Suggested Fix:**
Refactor to use a `@pytest.fixture` with `yield` (matching `test_hand_model.py`), or leverage the `conftest.py` autouse fixture and shared `SessionLocal`.

**Impact:** Purely cosmetic — tests work correctly and are properly isolated. Converging on the fixture pattern would improve consistency.
